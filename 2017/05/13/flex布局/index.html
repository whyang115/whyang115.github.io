flex布局

发表于 2017-05-13 | 分类于 笔记总结 |

flex布局意为弹性布局,是W3C提出的新一代布局方案,相比较传统布局方案(盒模型、float、position)而言,flex布局可以更高效、简便的实现各种布局

基本概念

  • 弹性容器: display属性被指定为flex | inline-flex(行内弹性容器)的元素自动成为弹性容器
  • 弹性元素(item): 弹性容器中的元素自动成为弹性元素
  • 主轴: 弹性容器的主轴是指弹性元素主要沿着哪个方向排列
  • 交叉轴: 主轴顺时针旋转90度即为交叉轴

弹性容器的属性

  • flex-direction: 定义主轴的方向
    • row(默认值): 主轴为水平方向,从左向右排列
    • row-reverse: 主轴为水平方向,从右向左排列
    • column: 主轴为垂直方向,从上到下排列
    • column-reverse: 主轴为垂直方向,从下到上排列

  • flex-wrap: 定义元素在一条轴线排列不完是如何换行

    • nowrap(默认值): 不换行
    • wrap: 换行,第一行在上方,按顺序排列
    • wrap-reverse: 换行,第一行在下方,按倒序排列

  • flex-flow: flex-direction和flex-wrap的缩写形式,默认值为 row nowrap

  • justify-content: 定义元素在主轴的对齐方式

    • flex-start(默认值): 按照主轴的方向排列
    • flex-end: 按照主轴的方向的反方向排列
    • center: 居中排列
    • space-between: 两端对齐,元素之间的距离相等
    • space-around: 元素两侧的距离相等,即元素之间的距离是元素与边框距离的两倍
  • align-items: 定义元素在交叉轴上的对齐方式
    • stretch(默认值): 若元素未设置高度,占满整个容器的高度
    • flex-start: 在交叉轴的起点对齐
    • flex-end: 在交叉轴的终点对齐
    • center: 在交叉轴的中点对齐
    • baseline: 在元素的第一行文字的基线对齐
  • align-content: 定义多根轴线的对齐方式,若只有一根轴线,此属性无效
    • stretch(默认值): 轴线占满整个交叉轴,
    • flex-start: 与交叉轴的起点对齐
    • flex-end: 与交叉轴的终点对齐
    • center: 在交叉轴的中点对齐
    • space-between: 与交叉轴的两端对齐,轴线之间的距离相等
    • space-around: 与交叉轴的两端对齐,轴线两侧的距离都相等

弹性元素的属性

  • flex-grow: 定义项目的放大比例,默认为0即如若存在剩余空间也不放大,当有多个项目时,按照定义的值等比例放大
  • flex-shrink: 定义项目的缩小比例,默认为1,当设为0时不缩小,当有多个项目时,按照所定义的值等比例缩小
  • flex-basis: 定义在分配多余空间之前,元素占据的主轴空间,默认值为auto即元素的原本大小,可设为一个固定数值使得元素的大小固定
  • flex: 是flex-grow、flex-shrink、flex-basis的缩写形式,默认值为 0 1 auto
  • order: 定义项目的排列顺序,数值越小越靠前
  • align-self: 定义元素自身在交叉轴上的对齐方式,可覆盖容器上定义的对齐方式

兼容性

flex布局对IE的兼容性并不是很好,但是这种布局方式在较为高级的浏览器上会是很常用的
